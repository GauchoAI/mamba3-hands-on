# jepa/ — JEPA-Cortex experiment

A small Mamba-3 byte-level LM that **talks** (English + Spanish, learned
from a Qwen-2.5-1.5B teacher) and **reasons** (via residual-stream
primitives like CounterPrimitive). Uses a JEPA-style thought-distillation
loss + a Cramér-Wold isotropy regularizer (SIGreg) to converge faster
than vanilla pseudo-label distillation at the same parameter count.

## Why a subfolder

This experiment ships with **its own copy** of `cortex_counting.py`,
`mamba3_minimal.py`, and the SSM scan helpers (`ssm_triton*.py`,
`ssm_scan_native.py`). Edits to `CortexLM.forward` (specifically a
`return_jepa=True` flag) live only on the local copy here — top-level
experiments are unaffected.

If a top-level fix needs to land in this experiment's copy, refresh by
hand:

```bash
cp cortex_counting.py jepa/cortex_counting.py   # then re-apply return_jepa edit
cp mamba3_minimal.py  jepa/mamba3_minimal.py
cp ssm_triton.py ssm_triton_kernel.py ssm_scan_native.py jepa/
```

## Layout

| File | Role |
|---|---|
| `make_teacher_thoughts.py` | Run Qwen-2.5-1.5B; write `data/teacher_thoughts.{bin,idx}` (paired text + last-layer hiddens at every 16 bytes of response) |
| `data_loader.py` | Memmap reader + collator + bilingual / counting iterators for the trainer's mixin |
| `arch.py` | `ThoughtHead`, `jepa_loss`, `sigreg_loss` — stateless modules that ride on top of `CortexLM` |
| `cortex_counting.py` | Local copy with `return_jepa=True` path |
| `mamba3_minimal.py` | Local copy of the Mamba-3 block |
| `ssm_triton.py`, `ssm_triton_kernel.py`, `ssm_scan_native.py` | Local copy of the SSM scan kernels (Triton on CUDA, JIT fallback on MPS/CPU) |
| `checkpoint.py` | Async, layered (light/heavy/best) checkpointer with atomic writes and a manifest log |
| `eval_daemon.py` | Watches `MANIFEST.jsonl` for new checkpoints; computes byte CE / counter accuracy / intent variance; serves a tiny dashboard on `:8080` |
| `train.py` | Joint trainer: byte CE + JEPA + SIGreg + primitive aux. Mixes teacher / bilingual / unary batches |
| `talk.py` | REPL over a trained checkpoint; hard-gates inference on any CounterPrimitive |

## Running locally (smoke)

`uv run` makes use of the project's `pyproject.toml` and isolates the env
without needing manual `.venv` activation.

```bash
# Optional deps for this experiment (transformers etc) install once:
uv pip install transformers accelerate sentencepiece

# Forward + loss smoke (CPU is fine):
uv run python -c "
import sys; sys.path.insert(0, 'jepa')
import torch, arch, cortex_counting
cfg = cortex_counting.CortexLMConfig(n_layers=2, d_model=64, max_seq_len=64, vocab_size=256)
m = cortex_counting.CortexLM(cfg)
toks = torch.randint(0, 256, (4, 32))
plens = torch.tensor([8]*4)
logits, prim, residual, intent = m(toks, return_jepa=True, prompt_lens=plens)
print('forward ok:', logits.shape, residual.shape, intent.shape)
print('sigreg(rand)=', float(arch.sigreg_loss(torch.randn(32, 64))))
"
```

```bash
# 0. on the local machine: commit + push
git add jepa/ data/README.md DEPLOYMENT.md && git commit -m "..." && git push

# 4. on the local machine: visit http://localhost:8080 (SSH tunnel forwards 8080)
```

## How JEPA-Cortex differs from Path A

Path A (`make_teacher_corpus.py` → `train_bilingual_mlx.py`) trains the
student on teacher-generated bytes via plain next-byte CE. Convergence is
slow because the supervision signal is one categorical per byte and the
student inherits the teacher's verbosity.

JEPA-Cortex adds two losses on the same byte stream:

- `jepa_loss` — at every 16 bytes of teacher response, the student must
  predict (in latent space) what the teacher's hidden state will be 16
  bytes later. Dense, low-variance signal that's invariant to surface
  verbosity, so the student can be terser than the teacher and still
  match.
- `sigreg_loss` — Cramér-Wold isotropy on the student's prompt-pooled
  intent embedding. One scalar λ; prevents the latent from collapsing
  to "always say what the teacher said."

The CounterPrimitive (and any future primitive) rides the residual the
same way as in `cortex_counting.py`. Hard-gates inference is enabled in
`talk.py` automatically — same property that gave the standalone Cortex
its 16× OOD generalization.
